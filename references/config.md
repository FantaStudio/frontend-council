# Configure members

Edit `council.config.yaml` to set chairman and members. See `council.config.example.yaml` for the default frontend-council setup.

```yaml
council:
  chairman:
    role: auto
  members:
    - name: junior-portal
      level: junior
      project: corporate-portal
      tier: fast
      model: composer-2.5-fast
      persona_file: references/personas/junior-portal.md
      emoji: 'рЯМ±'
      color: 'GREEN'
  settings:
    execution: desktop
    exclude_chairman_from_members: true
    timeout: 180
```

## Execution mode

- `settings.execution: desktop` (default) вАФ chairman launches **Task subagents** in Cursor Desktop with each member's `model`; responses are written via `council.sh record`.
- `settings.execution: cli` вАФ headless workers (`agent -p`, `claude`, `codex`, вА¶). The `start --cli` flag overrides to CLI.

## Member fields

- `name` вАФ stable identifier (lowercase, short), e.g. `junior-portal`.
- `level` вАФ seniority label for synthesis (`junior`, `middle`, `senior`); included in `results --json`.
- `project` вАФ project context slug (`corporate-portal`, `ai-platform`, `custom-solutions`); included in results.
- `tier` вАФ tier for model mapping (`fast`, `medium`, `pro`); maps via `tier-models.js`.
- `model` вАФ model id for Cursor Task (`composer-2.5-fast`, `gpt-5.5-medium`, `gemini-3.1-pro`); included in `start --json` and `results --json`.
- `command` вАФ only for `execution: cli`; generated by `generate-config.js` on install.
- `persona_file` вАФ path to persona markdown relative to skill root; assembled into per-member prompt at job start.
- `emoji` and `color` вАФ optional UI labels for checklist and status output.

Add custom members by appending entries to `members`. Each member must have a `persona_file` pointing to a markdown file with role context and response format.

Note: `install.sh` runs `generate-config.js`. In desktop mode, `command` is not required вАФ `tier`/`model` is enough.
